# Bot Insights Conversation Examples

These are synthetic, intentionally idealized worked conversations. They show
what a user can accomplish with the `bot-insights` skill when relevant Bot
Insights data is available, and they emphasize the target analysis flow over
tool friction.

They are not transcripts from live customer sessions. The SQL, script
invocations, and artifact shapes are meant to be realistic for the deployed
skill, but the conversations are curated so readers can see the investigation
pattern clearly.

## Examples

| Example | What it demonstrates |
| --- | --- |
| [Anomaly investigation](anomaly-investigation.md) | Incident-style bot-share spike triage with coincident verified and unverified movers. |
| [Cross-window capacity plan](cross-window-capacity-plan.md) | Planning analysis across non-adjacent windows that do not fit predefined reports. |
| [SIEM unavailable triage](siem-unavailable-triage.md) | A less-clean investigation where posture data is available but SIEM/security evidence is missing. |
